# 配色・視覚エンコーディング（HTML 補足ページ）

spec-writer の HTML 補足ページ（サマリー / 概況 / 比較・対比 / 配色で意味を伝える表）の **配色の使い方** と **コントラスト・装飾の設計指針**。HEX 全量は [`dads-tokens.md`](./dads-tokens.md) (DADS v2.0.1) を SSOT として参照する。本ファイルは「どの階調を何に使うか」のレシピに専念する。

出典: デジタル庁「ダッシュボードデザインの実践ガイドブック」(2026/3/31 版) のうち、仕様書文書に転用できる原則を抜粋。配色パレットは DADS デザイントークン v2.0.1 と完全に同じ HEX 体系を採用する。

## カラーパレット（DADS 準拠の主要階調）

DADS は **10 色族 × 13 階調** を提供する（Blue / Light Blue / Cyan / Green / Lime / Yellow / Orange / Red / Magenta / Purple）。本ファイルでは視覚エンコーディングで実際に使う頻度の高い「主要 4 階調」のみ示す。**全階調 HEX は [`dads-tokens.md`](./dads-tokens.md) §2 を参照**。

背景色 `#FFFFFF` を 1 とした時のコントラスト比が右に行くほど強くなる。**コントラスト比 3:1 を満たす階調を選ぶ**（500 以上が UI 非テキストの境界、700 以上がテキストの境界の目安）。

## 図表タイトルの命名

図・表・グラフのタイトルは「何のデータか」を誤認させないことを最優先する。

- **フォーマット**: `[対象] の [指標名]（[データ種別]）`
  - 例: 「国産自動車の出荷台数（月次推移）」「都道府県別の人口（2024 年時点）」「カテゴリ別売上の構成比（前年比）」
- **Do**: 内容とデータ種別（月次推移 / 累計 / 構成比など）を明記する
- **Don't**: 「売上」「会員」のような名詞だけの曖昧なタイトル、逆に一文で説明しきろうとする冗長なタイトル

## コントラスト比のルール

DADS / JIS X 8341-3:2016 AA 準拠:

1. **テキスト（本文・リンク等）: 4.5:1 以上**
2. **UI 非テキスト（ボタン枠・データ色面等）: 3:1 以上**
3. 3:1 を満たせない色面: 色面領域の **すぐ近くに数値を記載**（その数値色は背景に対し 4.5:1 以上）
4. 数値の近接表示も無理な場合: マウスオーバーまたはキーボードフォーカス時に数値表示
5. Blue パレットを基準色にする場合、色面は `Blue 500` (`#4979f5`) 以上の濃さを使う

## 色覚多様性への配慮

- カラーユニバーサルデザイン推奨配色を使う
- 色覚特性シミュレーター / チェッカーで検証
- **色以外の識別方法を併用する**:
  - データ点の形（丸・四角・三角）
  - 塗り模様（網掛け・斜線）
  - ラベル直接付与
- 白黒印刷でも伝わるかを基準にする

## 装飾の排除（情報密度を上げる）

- 3D 表現を **使わない**
- ドロップシャドウを **使わない**（カード境界の subtle なものは可、装飾としては不可）
- 過剰なグリッド線を **使わない**（最小限に）
- 数値と直接関係のないテキストは **薄い色**（Solid Gray 500/700）にする
- 情報量を必要最低限に絞る
- 数値を不要に小数点まで表示しない（`3,000.00` ではなく `3,000`）

## 色数の制約

- 使用する色数は **1〜5 色程度に絞る**
- 注目すべき要素を強調色、それ以外をニュートラル色（Solid Gray）にする
- 多色の場合も色覚多様性配慮を必ず適用

## 状態の色分け（正常 / 警告 / 異常）

数値の状態を色で表現するときの推奨パターン（DADS セマンティック準拠）:

| 状態 | 推奨色 | HEX | 例 |
|---|---|---|---|
| 正常 | Solid Gray 500 または無地 | `#7f7f7f` | 通常進捗 |
| 強調（注目させたい主指標） | Blue 600 | `#3460fb` | 全体進捗率 |
| 警告（閾値接近） | Orange 600 | `#fb5b01` | 予算消化率 80% 超 |
| 異常 / 遅延 / 不適合 | Red 800 | `#ec0000` | 工程遅延、予算超過、エラー発生 |
| 達成 / 成功 | Green 600 または Green 800 (テキスト) | `#259d63` / `#197a4b` | 目標達成、KPI クリア |

**重要**: 色だけで状態を識別させない。**形 (▲ / ⚠) / アイコン / ラベル / 数値併記** のいずれかを必ず併用する（WCAG 1.4.1、色覚多様性配慮）。閾値の境界値（例: 「2,000 以上で赤、未満でグレー」）は凡例で明示する。
