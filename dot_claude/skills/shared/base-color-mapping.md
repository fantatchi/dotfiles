# ベースカラー切り替えと階調マッピング（共通リソース）

複数スキル（`spec-design` / `dashboard-design`）から参照される **ベースカラー運用ルール** の共通出典。HTML 補足ページ / PDF / スライド / ダッシュボードの配色選定で使う。

**位置付け**: 本ファイルは `~/.claude/skills/shared/` 配下のライブラリ（`name:` 付きスキルではない、自動起動対象外）。配色パレットの HEX 値の正本は `~/.claude/skills/dashboard-design/references/visual-encoding.md` の「## カラーパレット（デジタル庁デザインシステム由来）」セクションを参照。本ファイルはその HEX 値を **「どう使い分けるか」のマッピングルール** に専念する。

## 1. デフォルトのベースカラー: Blue（青）

特に指定が無い新規プロジェクトでは **Blue 系列** を採用する。

優先順位:

1. **プロジェクト固有指定がある場合**（CLAUDE.md / 既存仕様書のスタイル / ブランドガイド等）→ それを最優先で踏襲
2. **指定がない・新規プロジェクト・既存スタイル無し** → Blue 系列をデフォルト採用

ベースカラーを Blue 以外（Green / Orange / Light Blue / Cyan / Red）に変える場合は、後述の階調マッピングルールに従う + プロジェクトの ADR に「ベースカラー選定」を残す（`spec-design/references/adr-format.md` の ADR テンプレを参照）。

## 2. 階調番号と用途の対応（基本ルール）

**階調番号（50 / 200 / 400 / 600 / 900 / 1200）を維持したまま色相だけ切り替える** のが基本。用途と階調番号の対応は以下を守る:

| 用途 | 階調番号 | 役割の趣旨 |
|---|---|---|
| リンク / ハイライト | **900** | 視認性高め・本文上で目立つ濃さ |
| チャート Primary（主要指標） | **1200** | パレット内で最も濃い、主役の表現 |
| Positive（強調・正常達成） | **600** | コントラスト比 3:1 を満たす中濃度（※色相依存で破綻するケースあり、§4 を参照） |
| ラベル / 控えめ要素 | **200** | 装飾的・サブ要素 |
| 背景アクセント | **50** | ごく薄い面塗り、ノートカード等 |

## 4. コントラスト比マトリクス（重要 — 600 階調は色相依存で破綻する）

「600 階調 = 3:1 を満たす」という前提は **色相によっては成立しない**。Standard 背景 `#F8F8FB` に対する 600 階調のコントラスト比:

| 色相 | 600 HEX | vs `#F8F8FB` | 判定 |
|---|---|---|---|
| Blue 600 | `#3460FB` | 5.39:1 | ✅ 余裕あり |
| Light Blue 600 | `#008BF2` | 3.31:1 | ⚠️ 3:1 ギリ |
| **Cyan 600** | `#00A3BF` | **2.83:1** | ❌ **3:1 未達** |
| Green 600 | `#259D63` | 3.27:1 | ⚠️ 3:1 ギリ |
| **Orange 600** | `#FB5B01` | **3.00:1** | ⚠️ **3:1 ジャスト・余裕ゼロ** |
| Red 600 | `#FE3939` | 3.69:1 | ✅ |

**運用ルール**:

- **Cyan / Light Blue / Green / Orange を 600 で Positive に使う場合は要注意**。アイコン併用 / 数値併記 / 拡大表示 のいずれかで補強する
- 文字色として 600 階調を使う場合は背景に対し **4.5:1 以上**（WCAG AA 本文基準）を別途確認する。多くの色相で 600 は本文文字色には不向き、リンクや強調表示等の限定用途で使う
- 確実に 3:1 を超えたい場合は **900 階調** を採用する（全色相で安全）

## 5. 切り替え時に変えないもの（共通固定）

以下は **ベースカラーに関わらず固定** とする。色覚多様性とアクセシビリティの一貫性確保のため:

| 要素 | 固定値 | 理由 |
|---|---|---|
| Negative / Error / 警告 | Red 600 `#FE3939` / Red 900 `#CE0000` | 赤系で固定（universal な警告色） |
| Solid Gray 全階調 | `#F2F2F2` 〜 `#1A1A1A` | ニュートラル要素として共通利用 |
| テキスト本文 | Black `#000000` / Label `#626264` | コントラスト確保 |
| 背景の基本 | White `#FFFFFF` / Standard `#F8F8FB` | 共通 |

### 重要: Negative=Red 固定の副作用 — 面塗り禁止 MUST

ベースカラーが赤系・赤近接の色相（Orange / Red ベース）または濃い緑（Green ベース）の場合、**Negative の Red 600 面塗りは可読性を破壊する**:

| ベース色相 | Red 600 (#FE3939) vs ベース 900 のコントラスト比 | 影響 |
|---|---|---|
| Green 900 (`#115A36`) | **2.31:1** | Negative の文字が読めない |
| Orange 900 (`#AC3E00`) | **1.70:1** | Negative とベースが同化 |

**ルール**:

- **Negative は文字色や背景の面塗りで使わない**: 必ず **アイコン（⚠ / ✕ / △）+ ラベル + 数値併記** の二重符号化を使う（MUST）
- 状態を色のみで識別させない（色覚多様性配慮）
- 面塗りで強い注意を引きたい場合は **ベースカラーの 50 階調背景 + 900 階調枠線 + アイコン** で代替する

## 6. ベースカラーの選定指針

| ベースカラー | 適する用途 | 注意点 |
|---|---|---|
| **Blue（デフォルト）** | 一般的な仕様書・技術文書・行政文書・教育 | 安全側、迷ったら Blue |
| **Light Blue / Cyan** | SaaS・データ系・分析ダッシュボード・医療（清潔感） | Cyan 600 は 3:1 未達、Positive 用途は要強調補強 |
| **Green** | 環境・農業・金融（プラス指標主役）・成功状態が主役・医療（健康・成長） | Negative=Red 面塗りと相性が悪い、§5 のルール厳守 |
| **Orange** | 注意喚起・物流・建設・警告系・エネルギー | Negative=Red と色相が近く混同リスク、二重符号化 MUST |
| **Yellow** | 注意・教育・子ども向け | パレット表に Yellow 800 `#A58000` あり、本文テキストとしては明度高すぎで非推奨、強調限定 |
| **Red** | 非推奨 | Negative と衝突する。医療・緊急通報など特殊用途のみ、その場合は Negative を別色にする ADR が必要 |

### 業種別の標準的選択

| 業種 | 推奨ベース | 理由 |
|---|---|---|
| 行政・官公庁・公共 | Blue | 標準・信頼性 |
| 医療・健康 | Blue（信頼）/ Green（健康） | Negative の赤と衝突しないものを優先 |
| 金融・会計 | Blue | 赤緑が「損益」と衝突するため Blue 推奨 |
| 教育 | Blue / Green | 学齢別に多色許容も可 |
| スポーツ | チーム色優先（本ガイド外） | ブランドガイドに従う |
| IT / SaaS | Light Blue / Cyan | データ系の慣例 |
| 物流・建設・製造 | Orange | 警告系業務との親和性 |

## 7. 運用: ADR への記録

ベースカラーを Blue 以外にした場合、または Blue を採用したが業種的に複数案があり迷った場合、プロジェクトの ADR に「ベースカラー選定」を 1 件残す。テンプレートは `~/.claude/skills/spec-design/references/adr-format.md` の「## ベースカラー選定 ADR テンプレ」セクションを参照。

最低限残すべき項目:

- 採用したベースカラー
- 採用した理由（業種 / ブランド / 既存スタイル）
- 棄却した候補と理由
- Negative の扱い（Red 固定 / 例外的に別色化した場合はその HEX）
