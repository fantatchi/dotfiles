# 汎用コーディング規約

## TypeScript / 型安全性

- `interface` ではなく `type` を使用する。やむを得ず `interface` を使う場合は `//MEMO:` でコメントを付ける
- `as`、`any`、Non-null assertion（`!`）は使用禁止（`as const` のみ許容）
- 更新関数で `Partial<Omit<T, 'xxxId'>>` を使用して ID の意図しない上書きを防ぐ

## React / コンポーネント設計

- 2025年の React ベストプラクティスに準拠
- Props 型の名前は常に `Props`
- props スプレッド構文（`{...props}`）は使用しない
- `boolean` の props で `={true}` は省略する（例: `<Comp disabled />` ）
- 条件付き props が複数並ぶ場合は children か専用コンポーネントへの分離を検討する
- アロー関数で統一する

## インポート / エクスポート

- `import` 文はファイル上部にまとめる
- 同一ファイルでしか使っていなければ `export` しない
- 関数内の動的インポートはしない

## 命名 / コメント

- 文字列をハードコードしない
- マジックナンバーを避け、定数定義する
- 不要なコメントは書かない。リーダブルコードに則る
- YAGNI 原則を徹底する

## 定数管理

- 定数の重複定義をしない
- 別用途の定数グループを「文言が同じだから」と流用しない
