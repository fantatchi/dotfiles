# グラフ選択ガイド

データの性質と伝えたい情報からグラフ種を決める。SKILL.md の「設計フロー Step 3」から呼び出される参照資料。

## グラフ種別マトリクス

| 伝えたい情報 | 推奨グラフ | 補足 |
|---|---|---|
| 時間変化・傾向 | 折れ線グラフ | 縦軸=数量、横軸=時間 |
| 数量比較（カテゴリ間） | 棒グラフ | 円より優先 |
| 数量+構成比の時間変化 | 積み上げ面グラフ | 単一系列なら折れ線で十分 |
| 構成比のみの時間変化 | 100% 積み上げ面グラフ | |
| 構成比（時点・コンパクト表示） | 円グラフ | 全体総量が明らかな時のみ |
| 構成比（多くの場合） | 棒グラフ | 円より正確に伝わる |
| 全体総量が時間変化する構成比 | 面グラフ | 円は不適 |

## 折れ線グラフ

- **横軸は時間である必要がある**。横軸が時間以外なら棒グラフを検討
- データ点に丸/四角などの図形を配置（補間の有無を明示）
- 系列が複数なら系列名を折れ線の近傍に配置
- 用途例: 二系列比較、参照値比較、目標値比較

## 棒グラフ

- 一つの軸に数量、もう一つの軸に分類
- カテゴリ間の数量比較に最適
- 棒と棒の間には**隙間を空ける**
- 集合棒グラフ: 分類ごとに複数の棒を並べる
- 積み上げ棒グラフ: 棒を積み上げて構成比を表現

## 面グラフ

- 折れ線グラフに塗りを加えたもの
- 数量と構成比の時間変化を同時に伝える
- 主に積み上げ面で使用
- **構成比を伝える必要がなければ折れ線で十分**
- 系列数が多い、または時間変化を伝えない場合は棒へ

## 円グラフ

- 全体総量が明らか + コンパクトに構成比を伝える場合に限定
- それ以外は**棒グラフの方が正確に伝わる**
- 全体総量の数値を併記する
- ドーナツグラフは円の中心に情報を書ける派生形
- 全体総量が時間変化する場合は面グラフへ

## グラフを選ぶ前の判断軸

1. データの軸は何か（時間 / カテゴリ / 連続値）
2. 何を比較するか（時系列推移 / カテゴリ間 / 全体に対する割合）
3. 系列数（1 / 2〜5 / 多数）
4. 全体総量は固定か変動か
