## コントラスト比のルール

1. **背景色 vs グラフ色面: 3:1 以上**を確保（アクセシビリティ準拠）
2. 3:1 を満たせない場合: グラフ色面領域の**すぐ近くに数値を記載**（その数値色は背景に対し 4.5:1 以上）
3. 数値の近接表示も無理な場合: マウスオーバーまたはキーボードフォーカス時に数値表示
4. Blue パレットを基準色にする場合、グラフ色は `Blue 500` 相当（`#4979F5`）以上の濃さを使う

## 色覚多様性への配慮

- カラーユニバーサルデザイン推奨配色を使う
- 色覚特性シミュレーター/チェッカーで検証
- **色以外の識別方法を併用する**:
  - データ点の形（丸・四角・三角）
  - グラフの塗り模様（網掛け・斜線）
  - ラベル直接付与
- 白黒印刷でも伝わるかを基準にする

## 装飾の排除（情報密度を上げる）

- 3D 表現を**使わない**
- ドロップシャドウを**使わない**
- 過剰なグリッド線を**使わない**（最小限に）
- 数値と直接関係のないテキストは**薄い色**にする
- グラフの情報量を必要最低限に絞る
- 数値を不要に小数点まで表示しない（`3,000.00` ではなく `3,000`）
- 凡例・タイトル・小数部などが必要以上に記載されないこと

## 色数の制約

- グラフに使用する色数は **1〜5 色程度に絞る**
- 注目すべき系列を強調色、それ以外をニュートラル色（Solid Gray）にする
- 多色の場合も色覚多様性配慮を必ず適用

## 凡例

- グラフと**隣接**配置
- 凡例の**順序**をグラフ内の系列順序と一致させる
- 凡例とグラフが離れていると対応関係が分からなくなる

## 異常値・閾値の色分け

数値の状態（正常 / 警告 / 異常）を色で表現するときの推奨パターン:

| 状態 | 推奨色 | 例 |
|---|---|---|
| 正常 | `Solid Gray 400` (`#999999`) または無地 | 通常進捗 |
| 強調（注目させたい主指標） | `Blue 600` (`#3460FB`) | 全体進捗率 |
| 警告（閾値接近） | `Orange 600` (`#FB5B01`) | 予算消化率 80% 超 |
| 異常 / 遅延 / 不適合 | `Red 600` (`#FE3939`) | 工程遅延、予算超過、エラー発生 |
| 達成 / 成功 | `Green 600` (`#259D63`) または `#197A4B` | 目標達成、KPI クリア |

**重要**: 色だけで状態を識別させない。**形 (▲ / ⚠) / アイコン / ラベル / 数値併記** のいずれかを必ず併用する（色覚多様性配慮）。

閾値の境界値（例: 「2,000 以上で赤、未満でグレー」）は事前にデータ定義で明示し、ダッシュボード上でも凡例で示す。
