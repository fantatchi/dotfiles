# タイトル・凡例・文脈の付け方

タイトル命名規則、凡例配置、比較対象の併記、メタ情報の扱い。SKILL.md「設計原則 4: タイトル・凡例で文脈を補う」から呼び出される。

## タイトル命名規則

### Do
- グラフの**内容**と**データ種別**を明記
  - 例: 「国産自動車の出荷台数（月次推移）」
- 数値の意味を正確に表す
- 月次推移、累計、構成比などのデータ種別を記載
- 見る人がデータを誤認しないようにする

### Don't
- 「売上」「国産自動車」のような**主語だけ・名詞だけ**の曖昧なタイトル
- 何を表すグラフかすぐに分からないタイトル
- 「有効な会員 ID を持っている人数のうちでプレミアムメンバーシップ制度に登録している人の比率の数値」のような**冗長な説明文**

### 命名のフォーマット例
```
[対象] の [指標名]（[データ種別]）
```
- 「国産自動車の出荷台数（月次推移）」
- 「都道府県別の人口（2024 年時点）」
- 「カテゴリ別売上の構成比（前年比）」

## 凡例の配置

- グラフと**隣接**させる（離れていると対応関係が分からない）
- 凡例の**順序**をグラフ内の系列順序と**一致**させる
  - 上から「シティホテル → ビジネスホテル」と見えているなら、凡例も同じ順序
  - 系列順序と凡例順序が逆だと誤読を招く
- 系列名は折れ線・棒の近傍にも配置できると望ましい

## ラベル・系列名

- シンプルかつ端的に
- 冗長な説明文にしない
- 前提知識を必要とする表現を避ける

## 比較対象の併記

数量単体では判断が難しいデータには、比較対象を提供する:

- **目標値** との差分
- **平均値** との差分
- **前年値** との差分（前年比 / 前年同月比）
- **参照値**（業界平均、ベンチマーク）

時系列グラフでは目標値線と実績値線を両方表示するなど、比較対象を視覚的に重ね合わせる。

## データ定義の参照可能化

何を対象としたデータか、数値の意味、いつ更新されたかを参照できるようにする:

- データ定義: 例「購買キャンセルなどを加味しない、速報値ベースでの売上」
- 単位: %, m, 円, 件 など
- 更新頻度: 月次、週次、日次
- 閾値の考え方: 例「数値が 2,000 以下と 2,000 以上で色分け」

## メタ情報の記載

ダッシュボード全体に以下を併記:

- データの**ソース**
- データの**更新日時**（「2024/8/26 時点」など）
- 注釈
- 免責事項

見る人によって解釈の幅が出ないよう補足情報を提供する。

## 並び順に意味を持たせる

棒グラフ・円グラフ・表の項目の並び順は意味を持たせる:

- **数量の大小順**（多い順 / 少ない順）
- **時系列順**（更新日順）
- **アルファベット順**（一般認知される並び順なら）
- 50 音順（都道府県をあいうえお順）は、比較目的なら**避ける**（数量大小順を優先）

## 強弱をつける

ダッシュボード内の重要な情報には他と異なる表現を用いて注意を喚起:

- 色（強調色 vs ニュートラル）
- サイズ（大きさ）
- 文字の太さ

注目してほしいグラフが何かを見る人が即座に判断できるようにする。
