# Phase 4: 判断メモ圧縮

`context.md` の `## 判断メモ` セクションを仕分けして定常サイズに保つ。`## 進行中の作業` の 14 日ローテーション（context-save が機械的に削除）とは異なり、判断メモは「消す前に昇格すべきか」を判断する必要があるため、機械削除ではなく **判断パス** として行う。

## 実行条件

- context-save が「⚠️ 判断メモが N 件（15 件超）」とアラートした
- または、このセッションでプロジェクトをまたいで再利用される普遍知見が生まれた

## 4 分類ルール

`## 判断メモ` の各エントリを次の 4 つに振り分ける。

| 分類 | 条件 | 行き先 |
|---|---|---|
| **昇格** | 恒久 + 普遍（プロジェクト/時間を超えて効く）+ 未カバー | MEMORY.md に新規ファイル（`feedback_*.md` / `reference_*.md` / `user_*.md`） |
| **統合** | 恒久だが既存 memory が同テーマを持つ | 既存 memory ファイルに追記 |
| **削除** | CLAUDE.md / docs / 既存 memory で既出、または時限的な雑事で陳腐化 | context.md から落とす |
| **残置** | 時限だが再トリガー条件付きで「生きている」判断 | context.md に残す |

## 手順

1. `{project-root}/.claude/context.md` の `## 判断メモ` を抽出し、トップレベル箇条書きを 1 エントリとして列挙する
2. 各エントリを 4 分類に振り分ける。判定根拠（どの memory / CLAUDE.md / docs に既出か）を必ず添える
3. **削除・統合の前に、既出と判定した実ファイルを Read で開いて本当にカバーされているか確認する**（誤削除より誤残存を選ぶ。CLAUDE.md「削除前に何を失うか検証」）
4. 昇格: MEMORY.md frontmatter 規約（`name` / `description` / `metadata.type`）に従って新規ファイルを作成し、`MEMORY.md` 索引に 1 行追加する。本文は `**Why:**` / `**How to apply:**` を付け、関連を `[[name]]` でリンク
5. 統合: 既存 memory に追記。重複させない
6. 削除・昇格・統合したエントリを context.md から除去し、残置のみを残す
7. 仕分け結果（昇格 N / 統合 M / 削除 K / 残置 L）をユーザーに提示する

## 注意

- **判断を要するため、ユーザー確認なしに昇格・削除を適用しない**（session-review 全体の共通ルールに従う）
- 視覚設計など「スキル自身に属する知見」が context.md に残っている場合は、当該スキルの `references/` への移設も選択肢として提示する
- context.md 本体の `## 判断メモ` 編集はリスクが高いので、削除エントリが多いときはセクション全体の置換より、残置エントリを Read で pristine に取得してから 1 回の Edit で置換する方が安全
